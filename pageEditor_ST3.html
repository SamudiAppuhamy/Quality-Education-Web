Page Editor - Student 3

Documentation of my contributions to the Quality Education website project, focusing on UX design, accessibility, and interactive elements.

Pages Created

As the UX specialist for the Quality Education website, I was responsible for designing and implementing the following pages:

  • Feedback Form - Comprehensive form with various input types
  • Team Page - Interactive team member cards with flip animation
  • Teacher Training - Content page about teacher training initiatives
  • Validation Page - Validation Page

Implementation Details

Feedback Form

The feedback form was designed to collect comprehensive user input while maintaining a user-friendly experience. Key implementation features include:

  • Organized fieldsets with clear legends for logical grouping of form elements
  • Diverse input types (text, email, checkbox, radio, select, date, etc.) to demonstrate form versatility
  • Client-side validation with visual error indicators
  • Custom styling for form elements to maintain design consistency
  • Emoji-based rating system for intuitive feedback collection
  • Character counter for text areas to guide user input
/* Form Validation Styling */
input:invalid,
textarea:invalid,
select:invalid {
    border-color: #d32f2f;
}

.error-message {
    color: #d32f2f;
    font-size: 0.875rem;
    margin-top: 0.5rem;
    display: none;
}

input:invalid + .error-message,
textarea:invalid + .error-message,
select:invalid + .error-message {
    display: block;
}

Team Page

The team page features interactive cards with a flip animation to showcase team members and their contributions. Implementation highlights include:

  • 3D card flip effect using CSS transforms and perspective
  • Accessible implementation with keyboard navigation support
  • Responsive grid layout that adapts to different screen sizes
  • Consistent styling with the global design system
  • Focus states for improved accessibility
/* Card Flip Animation */
.member-thumbnail {
    position: relative;
    width: 100%;
    height: 100%;
    border-radius: var(--border-radius);
    overflow: hidden;
    box-shadow: var(--shadow);
    transition: transform 0.6s;
    transform-style: preserve-3d;
}

.member-front, 
.member-back {
    position: absolute;
    width: 100%;
    height: 100%;
    backface-visibility: hidden;
    border-radius: var(--border-radius);
}

.member-back {
    transform: rotateY(180deg);
}

.team-member:hover .member-thumbnail,
.team-member:focus-within .member-thumbnail {
    transform: rotateY(180deg);
}

Team Page Accessibility

The flip card design on the team page presented a particular accessibility challenge. I addressed this by:

  • Making the cards keyboard focusable with tabindex="0"
  • Adding appropriate ARIA roles and labels
  • Ensuring the flip animation works with keyboard focus (:focus-within)
  • Providing sufficient color contrast on both sides of the card

Feedback Form Accessibility

For the feedback form, I implemented several accessibility enhancements:

  • Logical tab order following the visual layout
  • Clear error messages for validation failures
  • Required field indicators
  • Grouped related inputs using fieldset and legend
  • Added title attributes with additional instructions where needed

Technical Challenges

3D Card Flip Animation

Implementing the 3D card flip effect while maintaining accessibility was challenging. I addressed this by:

  • Using CSS 3D transforms with appropriate perspective settings
  • Ensuring both sides of the card were properly positioned
  • Adding keyboard accessibility through focus states
  • Testing across different browsers to ensure consistent behavior

Form Validation

Creating a form with client-side validation that was both functional and user-friendly required:

  • Balancing HTML5 validation with custom styling
  • Providing clear, helpful error messages
  • Ensuring validation didn't interfere with user experience
  • Making validation accessible to screen readers

Responsive Design for Complex Elements

Ensuring the team cards and form elements worked well on all screen sizes required careful consideration:

  • Adapting grid layouts for smaller screens
  • Ensuring form elements remained usable on mobile devices
  • Maintaining the functionality of interactive elements across devices
  • Testing on various screen sizes to verify the responsive behavior
