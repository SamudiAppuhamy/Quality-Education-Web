CSS Variables

I implemented CSS variables (custom properties) to make the design system flexible and maintainable:

:root {
    --primary: #c5192d; /* SDG 4 Red */
    --primary-dark: #a11526;
    --secondary: #4c9f38; /* SDG Green */
    --neutral-100: #ffffff;
    --neutral-200: #f5f5f5;
    --neutral-300: #e0e0e0;
    --neutral-400: #9e9e9e;
    --neutral-500: #616161;
    --neutral-600: #424242;
    --neutral-700: #212121;
    --font-primary: "Roboto", Arial, sans-serif;
    --font-secondary: "Open Sans", Arial, sans-serif;
    --shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
    --border-radius: 4px;
    --transition: all 0.3s ease;
}

This approach allowed all team members to easily maintain consistent styling across their pages by referencing these variables.

Implementation Details

Home Page

The home page was designed to provide an engaging introduction to the Quality Education initiative and guide users to key content areas:

  • Hero section with a clear value proposition and call-to-action
  • Card-based grid layout for content page links with visual previews
  • Key facts section highlighting important statistics about education
  • Responsive design that adapts to different screen sizes

Programs Table

The programs table page presents complex information in an organized, interactive format:

  • Custom table styling with alternating row colors for readability
  • Highlighted featured row to draw attention to key programs
  • Tooltip functionality for column headers to provide additional context
  • Responsive table design that works on mobile devices
  • Consistent visual indicators (check/cross marks) for program features

Higher Education Access Content Page

This content page builds on the established content template with some unique features:

  • Comparison table for traditional vs. innovative education models
  • Statistical cards to highlight key data points
  • Structured content with clear section hierarchy
  • Internal navigation system consistent with other content pages

Technical Challenges

Responsive Navigation

Creating a navigation system that worked well on both desktop and mobile devices was challenging. I addressed this by:

  • Using flexbox for the desktop navigation with appropriate spacing
  • Implementing media queries to adapt the navigation on smaller screens
  • Ensuring touch targets were large enough for mobile users
  • Testing the navigation across different device sizes

Table Responsiveness

Tables are notoriously difficult to make responsive. For the Programs Table, I implemented several techniques:

  • Custom table styling that allows horizontal scrolling on small screens
  • Prioritizing important columns for mobile viewing
  • Using appropriate text wrapping and spacing for readability
  • Adding visual cues to help users understand the table structure

Cross-Browser Compatibility

Ensuring consistent rendering across different browsers required careful attention to:

  • Using standardized CSS properties with appropriate fallbacks
  • Testing in multiple browsers during development
  • Addressing specific browser quirks with targeted CSS
  • Balancing modern features with broad compatibility
